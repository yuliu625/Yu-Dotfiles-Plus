# Project Structure

## Overview
项目结构是隐含的重要设计，会被长期使用并产生各种长远的影响。我基于约定，构建了常见项目的必要内容，以简化项目新建和重构的影响。

## Basic Content
### asserts
一般有2个自目录:
- documents: 存放各种文本说明，一般为markdown文本。
- figures: 相关的图片。

### src
可以不叫这个名字，但统一以一个多个包或文件夹进行管理。
#### 常见分多个包的情况
多个包会有各自的环境配置，以避免环境的各种影响。
- 数据处理:
为了分离和管理数据pipeline，数据处理部分独立进行版本控制。
- 主项目
- 数据分析: 
数据分析和可视化部分的代码不要在当前项目中进行分析，但可以独立记录。

### scripts
运行其他包的总程序，约定对增加开放，对修改关闭。完全记录所有操作过程，不考虑代码冗余。
- 基础的，执行其他包的最高主程序。
- 基础的，自动运行和管理其他子进程的管理器。
- 各种自定义脚本。

### tests
基于pytest的测试工程。
- data: 测试需要的数据。为方便，数据可以由python对象进行存储。
- fixtures: 测试所需的hooks和fixtures。
- integration_tests: 集成测试部分的代码。
- unit_tests: 单元测试部分的代码。
- conftest.py: pytest进行识别约定的配置文件。
